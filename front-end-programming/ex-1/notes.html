Exercise - 1

Challenges Faced

  • DOM Manipulation Complexity: Managing DOM updates dynamically, especially adding new rows with specific CSS class manipulations, required meticulous attention to detail to ensure correctness.
  • CSS Styling: Balancing inline CSS and ensuring dynamic addition of styles (like the highlight class) worked seamlessly without conflicts.
  • Google Font Integration: Dynamically loading a Google Font and ensuring it applied consistently to the table was a nuanced task that required debugging.
  • Browser Information Display: Extracting accurate browser details and ensuring real-time updates with window resizing posed some initial issues.
  • JSON File Handling: Moving from inline data to external JSON required understanding of asynchronous loading and ensuring data availability before table rendering.
  • Module Separation: Dividing JavaScript into modules (data-loader.js and table-renderer.js) highlighted challenges in structuring and managing dependencies.
  • CSV Parsing: Integrating PapaParse for CSV parsing and ensuring compatibility with AG Grid's expected data format took effort.
  • AG Grid Customization: Configuring AG Grid for advanced features like row highlighting, sorting, and filtering involved steep learning and extensive documentation review.

Improvements for Future Iterations

  • Enhanced Styling: Move away from inline CSS completely and adopt a modular CSS approach or frameworks like Tailwind or Bootstrap for easier scalability.
  • Code Modularity: Implement an ES6 module loader or bundler like Webpack to improve dependency management and code organization.
  • Data Management: Use a lightweight state management library (e.g., Redux or Zustand) to manage student data more robustly.
  • Responsive Design: Ensure that the table and all other elements are fully responsive for various device screen sizes.
  • Additional Features: Add more interactive features like pagination, search functionality, or data export (CSV/Excel) options to enhance usability.
  • Performance Optimization: Optimize the loading and rendering of data, especially when dealing with large CSV files, to prevent UI lag.
  • Error Handling: Implement robust error handling mechanisms to manage issues like missing or malformed JSON/CSV data gracefully.
  • Testing: Write unit and integration tests for key features to ensure stability and facilitate future enhancements.
  • Documentation: Add inline comments and maintain an external README file to document the project structure and usage.
  • Library Exploration: Explore alternatives to AG Grid for comparison, such as DataTables or Handsontable, to evaluate the best fit for requirements.

Takeaways

  • Incremental development allows for addressing challenges in manageable chunks and iteratively improving the application.
  • Separation of concerns (HTML, CSS, JS) is critical for scalability and maintainability in web projects.
  • Using libraries and tools (e.g., AG Grid, PapaParse) accelerates development but requires a steep learning curve.
  • Understanding browser behavior and compatibility is essential for creating a robust front-end experience.
