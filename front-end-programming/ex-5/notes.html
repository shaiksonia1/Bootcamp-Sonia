Chrome DevTools Exploration Notes

1. Console and Errors

  • Opening Chrome DevTools: Press F12 or Ctrl + Shift + I to open Chrome DevTools. Navigate to the Console tab to print debugging information and explore errors.
  • Using console.log(): 
    let num = 10;
console.log("The value of num is:", num);
    This will print: "The value of num is: 10" in the Console.
  • Triggering an Intentional Error: 
    let element = document.querySelector("#missingElement");
console.log(element);
    This will print null or throw an error like Uncaught TypeError: Cannot read property 'addEventListener' of null if the element doesn't exist in the DOM.

2. Network Activity

  • Loading a JSON or CSV File: 
    fetch('data.json')
    .then(response => response.json())
    .then(data => console.log(data));
    This fetches a JSON file dynamically.
  • Observing the Network Tab: In the Network tab, observe:
    • Status Codes: Look for status codes like 200 OK for successful requests or 404 Not Found for failed ones.
    • Headers: Check the Request and Response headers for details about the request.
    • Time Taken: Look at the Time column to see how long the request took to load.

3. Elements and Styles

  • Inspecting a Dynamically Added Table Row: Right-click on the page and select Inspect to open the Elements tab. Find and inspect the <tr> element added dynamically (e.g., from Exercise 1).
  • Modifying CSS on the Fly: In the Elements tab, click on the element and navigate to the Styles pane. Example: Change the background color by modifying the CSS rule. 
    background-color: yellow;
    The changes will be reflected in real-time on the page.

4. Performance Metrics

  • Using the Performance Tab: Go to the Performance tab in DevTools. Click the Record button (circle icon) and reload the page. The tool will record and display a timeline of the page's loading process.
  • Analyzing Page Load Time: After recording, the timeline will show key events, such as:
    • Navigation: When the page started loading.
    • Scripting: JavaScript execution time.
    • Rendering: How long it took to render the page.
    • Painting: Time to render visual elements.
    Identify bottlenecks or high-latency areas (e.g., long scripting times or slow resource loading).
  • Optimizing Bottlenecks: If you see long times for specific tasks, you can consider optimizing:
    • Minifying JavaScript files.
    • Deferring non-essential scripts.
    • Using image compression for larger images.
