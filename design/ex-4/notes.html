Exercise 4: Basics of Images

Thought Process

In this exercise, I explored various aspects of image handling in web development. I started by understanding the differences between JPEG, PNG, and SVG formats and their respective use cases.

Introduction

Images are a critical part of modern web design. To begin, I reviewed different image formats:

  • JPEG: Used for photographs due to its efficient compression.
  • PNG: Best for graphics with transparency.
  • SVG: Ideal for scalable, resolution-independent icons and illustrations.

I also focused on basic editing techniques, such as resizing, compression, and applying transparency.

Level 1: Basics of Image Handling

In this step, I added images to the webpage and styled them using CSS. Key tasks included:

  • Applying width and height styles to control image dimensions.
  • Using border-radius to create rounded corners for a modern look.
  • Experimenting with placeholder image services, like Lorem Picsum, to quickly add generic images for practice.

This step helped me get comfortable with basic image handling techniques.

Level 2: Styling and Overlays

Here, I focused on making images more visually appealing by:

  • Adding a full-bleed image as a page header for a bold, modern design.
  • Applying overlays using CSS gradients and semi-transparent colors to enhance readability of text over images.

These techniques helped me understand how to integrate images seamlessly into the design.

Level 3: Responsive and Optimized Images

In the advanced section, I explored ways to make images more efficient and responsive:

  • Used the <picture> element to serve different images for various screen sizes and resolutions.
  • Focused on lazy loading to optimize image loading and improve page performance.

These practices ensure that the webpage is optimized for both performance and visual quality across devices.
