Design Notes and Redesigns

Level 1: Annotating Problems

Indian Government Press Releases

Website: Indian Government Press Releases

Issues Observed:

  • Cluttered Layout: The page is overcrowded with content, making it difficult for users to focus on important information. The headlines are not prominent enough to stand out from the rest of the text.
  • Bad Contrast: Text and background colors are not well contrasted, making the text hard to read, especially for users with visual impairments.
  • Lack of Search Functionality: There is no intuitive way to search for press releases, making it cumbersome for users to find specific releases.

Suggested Improvements:

  • Consider adopting a clean, minimalist design like US Department of State, which has a structured and easy-to-navigate interface with a clear distinction between news releases and other sections.
  • Implement a better contrast ratio, following the Web Content Accessibility Guidelines (WCAG).
  • Incorporate a search bar for users to quickly find press releases by date, title, or category.

IITM CSE Faculty Site

Website: IITM CSE Faculty Site

Issues Observed:

  • Navigation Issues: Navigation between faculty pages is not smooth. Users have to go back to the main faculty page each time they want to see another member's profile.
  • Minimal Visual Appeal: The website uses very basic fonts and has a lack of engaging design elements.
  • Responsive Issues: The site does not display well on mobile devices, with overlapping content and misaligned sections.

Suggested Improvements:

  • Redesign with a more modern layout, similar to CMU's Computer Science Faculty Page, which has a clean, visually appealing design and intuitive navigation.
  • Make use of a responsive design, allowing for proper scaling and arrangement on mobile screens.

IIT B Talk

Website: IIT B Talk

Issues Observed:

  • Text Overload: The site has too much information crammed together in a single page. It’s overwhelming and hard for users to quickly absorb key details.
  • Poorly Structured Information: There is no clear hierarchy of content (e.g., important talks vs. archived talks), which makes navigation difficult.

Suggested Improvements:

  • Adopt a grid-based design, like TED Talks, where talks are neatly organized by categories, topics, and speaker names, with easy navigation between them.
  • Break content into smaller sections with clear headings and a prominent "Featured Talks" section.

Indian Railways Website

Website: Indian Railways

Issues Observed:

  • Poor Visual Hierarchy: The website is overloaded with text, and the most important elements (such as booking and train status) are buried under a lot of irrelevant information.
  • Slow Loading: The website contains a lot of heavy images and scripts, which slow down the load time significantly.

Suggested Improvements:

  • Adopt a cleaner design like Amtrak's website, where important features like booking and train schedules are easy to find and quick to access.
  • Optimize images and scripts to improve page load speed.

Level 2: Comparing Two Websites

Flipkart Mobile Phones Page vs Amazon Mobile Phones Page

Flipkart Website: Flipkart

Amazon Website: Amazon

Design Comparison:

  • Flipkart: Flipkart has a grid layout with a left sidebar, featuring a large number of filters, but this could be overwhelming. The search bar is prominent, and the page feels cluttered in comparison to Amazon's cleaner interface.
  • Amazon: Amazon uses a top navigation bar, which simplifies the structure. Filters are more minimal and accessible, allowing for a better user experience. The design feels more refined, with consistent spacing between elements.

Information Structure Comparison:

  • Flipkart: Flipkart uses a category-based filtering system, but there’s no clear distinction between different categories, making it harder for users to make a decision.
  • Amazon: Amazon uses dynamic filtering and allows users to quickly narrow down options based on multiple parameters, including brand, price, customer ratings, and more.

Suggested Improvements:

  • Flipkart could consider Amazon's simpler, more focused design to avoid overwhelming users with excessive information.
  • Both sites could improve user decision-making by implementing clearer product descriptions, comparison options, and customer reviews upfront.

State of Telangana vs State of Michigan

State of Telangana Website: Telangana

State of Michigan Website: Michigan

Design Comparison:

  • State of Telangana: The design feels outdated, with a dense, text-heavy layout. Important sections are not easily distinguishable, which makes the user experience less intuitive.
  • State of Michigan: Michigan’s site uses a modern design with large icons and a more accessible layout. The information is structured clearly with a top navigation bar and organized categories.

Suggested Improvements:

  • Telangana could adopt Michigan's layout by using larger, visually distinct icons, and a more minimalist approach for better accessibility and navigation.
