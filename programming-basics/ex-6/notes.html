Exercise 6: YAML Configuration with config

Level 1

Problem Description:

  • Install config and create a default.yaml configuration file.
  • Load and use configuration values in the program.

Exit Criteria: The program reads values from the YAML file.

Level 2

Problem Description:

  • Load the appropriate configuration based on the environment.

Exit Criteria: The program switches configurations based on the environment.

Level 3

Problem Description:

  • Use the configuration to dynamically adjust program behavior (e.g., logging levels).
  • Specify additional commands, functions, and files in the YAML configuration file. Dynamically load and execute functions (e.g., sin and cos) from specified files.

Exit Criteria:

  • Configuration drives program behavior dynamically.
  • New functionality can be added without modifying the old code.

Challenges I Faced

  • Understanding the structure and syntax of YAML configuration files.
  • Switching between environments seamlessly during development and production.
  • Implementing dynamic function loading based on configuration files.

Improvements

  • Documented the process of environment switching using the NODE_ENV variable.
