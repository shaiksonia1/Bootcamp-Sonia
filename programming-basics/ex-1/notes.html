Node.js Project Setup Documentation

Level 1

Problem Description:

  • Initialize a new Node.js project using npm init.
  • Add a start script to run a basic index.js file.
  • Add a dev script to run the same file with debugging output (DEBUG=* node index.js).

Exit Criteria:

  • Running npm start executes index.js.
  • Running npm run dev shows debugging output.

Level 2

Problem Description:

  • Install the chalk package and use it in index.js to style console outputs.
  • Add nodemon as a dev dependency to automatically restart the program during development.

Exit Criteria:

  • index.js outputs styled messages using chalk.
  • Changes to index.js automatically restart when using nodemon.

Level 3

Problem Description:

  • Add a prestart script to ensure node_modules is available before running the program.
  • Test by removing node_modules and ensuring npm start reinstalls dependencies successfully.

Exit Criteria:

  • The prestart script checks for node_modules and reinstalls dependencies if missing.

Challenges I Faced

  • Understanding how to use scripts like prestart effectively to check for missing node_modules.
  • Configuring chalk for proper message styling in the console.
  • Ensuring nodemon restarts automatically without manual intervention.

Improvements

  • Added comments in package.json to explain each script's purpose.
  • Used chalk to provide better color-coded log outputs for improved readability.
  • Optimized prestart script to run checks efficiently and reinstall dependencies only when necessary.
