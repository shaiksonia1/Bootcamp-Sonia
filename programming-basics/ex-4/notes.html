Exercise 4: Debugging with debug

Level 1

Problem Description:

  • Install the debug package and set up a debugging namespace.
  • Log messages using the debug function in a single module.

Exit Criteria: Debug messages are displayed when the DEBUG environment variable is set.

Level 2

Problem Description:

  • Extend the program by adding multiple modules with unique debug namespaces.
  • Use environment variables to selectively enable debugging for specific modules.

Exit Criteria: Debugging can be turned on or off for individual modules.

Level 3

Problem Description:

  • Combine debug with winston for structured logging and debugging.
  • Enhance the compute program to make it interactive.
  • Use both tools to log and debug complex program scenarios.

Exit Criteria: Debugging and logging work together seamlessly in the program.

Challenges I Faced

  • Understanding how to use namespaces effectively with debug.
  • Integrating debug and winston for consistent logging.
  • Managing environment variables for selective debugging.

Improvements

  • Documented the setup of debug and winston integration for future reference.
  • Created clearer debug namespaces to identify specific modules.
  • Improved program interactivity for better debugging scenarios.
