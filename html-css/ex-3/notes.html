Project Notes for Exercise 3: Classless CSS

Goal

The goal of this exercise was to style a webpage without using any classes or IDs. This forces you to rely on element selectors and default browser styling to manage the design.

Introduction

In this exercise, I explored how blogs and markdown content are often converted into HTML without the need for additional markup like classes or IDs. I also learned about **normalize.css** and its role in resetting browser styles. The exercise helped me understand how to style basic HTML elements with minimal CSS and without any class-based structure.

Level 3: Using Classless CSS Frameworks

For this step, I explored a few **classless CSS frameworks** like classless.de. These frameworks help to style elements without the need for classes or IDs. They provide a simple and clean base styling for most common HTML elements.

After experimenting with these frameworks, I chose one that provided a minimalist design and easy-to-use layout components. Here are the things I liked about it:

  • The styles were simple and worked well with basic HTML elements.
  • No extra complexity from classes or IDs, making it easy to focus on the content itself.
  • It included default styles for elements like blockquotes, lists, and headers, which saved a lot of time.

I used this framework to enhance the design of my blog post, ensuring the page remained clean, readable, and minimalistic.

Challenges Faced

While working on this exercise, I faced a few challenges:

  • Limited control: Without classes or IDs, it was harder to target specific elements for styling.
  • Less flexibility: Customizing individual elements (like adding specific colors to certain links) was trickier without classes.
  • Overriding default browser styles: Even with normalize.css, some default browser styles had to be manually adjusted to achieve the desired look.
