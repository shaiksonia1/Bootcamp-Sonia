1. Problem Understanding and Objective

The task involves creating an interactive widget that allows a biologist to search for diseases, explore their hierarchical relationships (using classifications like EFO or MONDO), and select a disease of interest. Once selected, the widget should return the diseaseâ€™s code (EFO or MONDO) and optionally include its description.

2. Key Concepts

  • Target: In drug discovery, a target is a molecule involved in a disease process, modulated to treat the disease.
  • EFO (Experimental Factor Ontology): A structured vocabulary describing experimental factors and relationships in biological studies.
  • MONDO (Monarch Disease Ontology): A comprehensive ontology integrating various disease classification systems into a unified framework.
  • Disease Hierarchy: Organizes diseases from general categories to specific ones, with potential multiple classifications.
  • API Key: A unique identifier for authenticating and authorizing access to APIs.

3. Tools and Technologies

  • EBI OLS: Provides access to biological ontologies via REST APIs.
  • JavaScript and HTML: Used for creating an interactive widget.
  • Backend/API Integration: Fetches and processes data from the OLS API.

4. Process of Implementation

4.1 Disease Search & Hierarchy Exploration

The widget enables the search and hierarchical exploration of diseases through the following steps:

  1. Set up the frontend interface with search bars and hierarchical navigation.
  2. Fetch disease data from the OLS API based on user queries.
  3. Display data as a tree structure for hierarchical navigation.
  4. Show EFO or MONDO codes and descriptions after selection.
  5. Enhance user experience with features like search-as-you-type.

4.2 Obtaining the API Key

To access the OLS API, register on the EBI OLS website, generate an API key, and store it securely using environment variables.

Acquiring the API key posed a challenge during the implementation process. Despite following the provided documentation, I was unable to locate the necessary key through the standard procedures. To overcome this, I resorted to advanced troubleshooting techniques, including inspecting network requests via browser developer tools. This method allowed me to identify the key embedded in the network responses and successfully use it for API integration. While unconventional, this approach ensured seamless interaction with the OLS API.

4.3 Challenges Faced

  • Understanding the OLS API documentation.
  • Handling large datasets efficiently.
  • Dealing with API rate limiting.
  • Addressing disease ambiguity and missing descriptions.

5. Optional Features

  • Support for multiple ontologies.
  • Interactive hierarchy with expand/collapse functionality.
  • Detailed disease information display.

6. Demonstrating the Widget

The widget integrates into a web app featuring a search field, hierarchical display, and disease code/description output.

7. Conclusion

This widget simplifies disease exploration for researchers, enabling intuitive navigation of ontologies like EFO and MONDO. Despite challenges, features like real-time search and lazy loading ensure usability and performance.
